# 📒 単語帳アプリ

## ■ アプリ概要

プログラミング学習中に調べた専門用語や言語仕様を記録し、
効率よく復習できるように設計したWebアプリケーションです。

タグ・お気に入り・部分一致検索に加え、
条件に応じたランダム出題機能を実装しています。

出題順制御にはセッションベースのデッキ方式を採用し、
重複出題を防ぐ設計にしています。

---

## ■ 制作背景

学習を進める中で、一度検索した専門用語を
「後からもう一度見返したい」と感じる場面が増えました。

しかし、ブラウザ履歴やメモ帳では整理が難しく、
体系的に管理・復習できる仕組みが必要だと考え、本アプリを制作しました。

単なるCRUDアプリではなく、
**学習効率を高める設計**を意識しています。

---

## ■ 設計意図

### 1. 学習導線を止めないUI設計

検索後すぐに単語を登録できるよう、
単語追加フォームを画面上部に配置しました。

「調べる → 登録する → 復習する」までを最短導線にすることで、
学習フローを中断させない設計としています。

---

### 2. 探しやすさを重視した検索設計

- 部分一致検索（term / reading / meaning / note）
- タグによる絞り込み
- お気に入り機能

単語数が増えても目的の用語に即座にアクセスできる構造を意識しました。

---

### 3. 理解重視の出題形式

一般的な選択式ではなく、

> 問題 → 自分で思い出す → 答えを表示する

という形式を採用しました。

消去法で正解できてしまう形式ではなく、
**「意味を説明できるか」を重視した設計**にしています。

## ■ ER図

### テーブル構成

- wordbooks（単語帳）
- words（単語）
- tags（タグ）
- tag_word（中間テーブル）

### リレーション

- wordbook : words → 1対多
- words : tags → 多対多（tag_word）

中間テーブルにはユニーク制約を設定し、
同一タグの重複登録を防止しています。

---

## ■ 主な機能

### 単語管理
- 単語の作成 / 編集 / 削除
- タグの複数付与
- 新規タグの同時作成
- お気に入り登録

### 検索・絞り込み
- 部分一致検索
- タグ別絞り込み
- お気に入りのみ表示
- 並び替え（新しい順 / 古い順）

### 出題機能
- 全単語出題
- お気に入りのみ出題
- タグ別出題
- 現在の検索条件を維持した出題

---

## ■ 出題ロジックの設計

### デッキ方式を採用

出題のたびにランダム取得するのではなく、

1. 条件に合致する単語IDを取得
2. ID配列をシャッフル
3. セッションに保存
4. 1問ずつ取り出す

というデッキ方式を実装しています。

### 工夫点

- 検索条件・タグ・お気に入り条件をハッシュ化
- 条件ごとにデッキを分離管理
- プール変更時のみ再生成
- 空デッキ時の安全処理

これにより、
同一条件下での重複出題を防止しています。
